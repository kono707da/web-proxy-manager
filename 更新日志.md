## v1.4.2 - 2026-07-10

### 修复

- **start.bat PATH 设置报错「此时不应有 \NVIDIA」**：在 `if (...)` 括号块内使用 `%PATH%` 展开，若系统 PATH 含括号路径（如 `C:\Program Files (x86)\NVIDIA Corporation\...`）会破坏 bat 括号匹配导致语法错误。

## v1.4.1 - 2026-07-10

### 修复

- **启动脚本端口占用检测误判**：`kill_stale_port.ps1` 无法识别本项目进程（`python run.py`），导致本应用占用 8000 端口时误报为「其他程序占用」并拒绝启动。
  - 根因：`-like` 通配符匹配模式 `*backend\run.py*` 无法匹配 `python.exe" run.py`（run.py 前是空格而非反斜杠路径）
  - 修复：改用 `-match` 正则匹配，覆盖 `proxy_manager.main:app`、`\run.py`、`/run.py`、` run.py`（空格+run.py）、`uvicorn --port 8000` 五种命令行特征
  - 优化：`Get-NetTCPConnection` 返回多条（IPv4/IPv6）时按 PID 去重，避免重复查询和空 PID 条目干扰

## v1.4.0 - 2026-07-10

### 变更

- **Docker 构建不再下载 mihomo，改由容器运行时下载到挂载卷**：解决每次镜像重建都要从 GitHub 下载 mihomo 内核导致构建慢的问题。
  - Dockerfile 移除 mihomo 下载层及 `MIHOMO_VERSION`/`MIHOMO_ARCH`/`HTTP_PROXY` build-arg，构建阶段完全不访问 GitHub
  - 后端 `MihomoManager` 启动时新增 `_ensure_binary()`：优先用 PATH 中的 mihomo；找不到则检测 `work_dir`（挂载卷内）是否已缓存；都没有则从 GitHub 下载 gz 解压到 `work_dir/mihomo`（Linux），持久化到 Docker volume，容器重建不丢失
  - 新增配置项 `mihomo.binary_version`（默认 `1.19.28`）控制运行时下载版本
  - Jenkinsfile 移除 `--build-arg HTTP_PROXY`，构建无需代理
  - 下载通过 httpx，自动读取 `HTTP_PROXY`/`HTTPS_PROXY` 环境变量，部署服务器网络不通 GitHub 时可设代理
  - Windows 环境（本地开发）仍需手动放置 `mihomo.exe` 到 `backend/bin/` 或 PATH，不自动下载

## v1.3.3 - 2026-07-10

### 修复（根因修复）

- **Switch 开关不生效（所有页面）**：根因是 Switch 组件内部 `SwitchRoot` 使用了 `v-model:checked` 绑定，但 reka-ui 的 `SwitchRoot` 使用的是 `modelValue` / `update:modelValue`，`checked` 属性根本没被 `SwitchRoot` 识别。导致 Switch 进入非受控模式，内部状态自管理，切换页面后组件重建读取真实数据就还原。
  - 修复：`SwitchRoot` 改用 `v-model`（映射到 `modelValue`），`defineModel('checked')` 对外接口不变
  - 影响范围：SubscriptionsView、RulesView、LimitsView、QuotasView、DevicesView、ConnectionsView 所有 Switch
- **SubscriptionsView 表格恢复 Switch**：上一版本临时改为 Button，现在根因修复后恢复为 Switch

## v1.3.2 - 2026-07-10

### 变更

- **未注册设备默认走直连**：最终兜底规则从 `MATCH,PROXY` 改为 `MATCH,DIRECT`。未在设备管理中注册的设备，所有流量（包括国外）均走直连；仅已注册设备通过 `SRC-IP-CIDR` 规则走分配的代理节点。

## v1.3.1 - 2026-07-10

### 修复

- **订阅开关不保持（根因修复）**：reka-ui Switch 的 `defineModel('checked')` 在 API 调用失败时不会同步回旧值，导致开关视觉已切换但数据未保存。表格内改用 Button + Badge 替代 Switch，完全由数据驱动，杜绝状态不一致。
- **后端 reload 异常导致 API 500**：所有写操作（订阅增删改、设备增删改）的 `get_manager().reload(db)` 包裹 try-except，reload 失败仅记日志不影响已提交的数据返回。这是开关 API 返回 500 从而前端认为操作失败的根因。
- **订阅代理更新支持自定义代理 URL**：新增 `custom_proxy` 参数，用户可直接指定代理地址（如 `http://1.2.3.4:8080`），解决「需要代理才能访问订阅源但还没有可用节点」的鸡生蛋问题。
- **设备管理允许添加本机**：移除 discover 对 127.0.0.1 的排除，新增「添加本机」按钮，可将服务器自身纳入设备管理。

### 变更

- 订阅管理工具栏：「使用代理更新」改为「走内核代理」，新增自定义代理 URL 输入框
- 订阅更新 API（`/api/subscriptions/{id}/update`、`/api/subscriptions/update-all`）新增 `custom_proxy` 查询参数

## v1.0.0 - 2026-07-09

### 首次发布

基于 [mihomo (Clash.Meta)](https://github.com/MetaCubeX/mihomo) 内核的 Linux Docker 代理管理器，参考 [clash-verge-rev](https://github.com/clash-verge-rev/clash-verge-rev) 设计，通过 Web UI 进行流量控制。

### 新增功能

#### 后端（Python + FastAPI）

- **配置加载**：三层配置（config.yaml → 环境变量 `PROXY_MANAGER_` 前缀 → settings.json），敏感 secret 持久化到 data 目录
- **数据库**：SQLAlchemy 2.0，6 张表（User / Subscription / Rule / SpeedLimit / TrafficQuota / TrafficStat），首次启动自动创建默认管理员 admin/admin123
- **认证**：bcrypt 密码哈希 + JWT (python-jose)，手写 Header Bearer 依赖，支持 token 刷新
- **mihomo 内核管理**：
  - `MihomoClient`：封装 version / configs / connections / proxies / rules RESTful API
  - `config_builder`：订阅拉取与缓存、proxy-groups（select + url-test 自动测速）、限速写入（download-speed-limit / upload-speed-limit）、配额阻断规则（IP-CIDR,x.x.x.x/32,REJECT）自动注入、默认规则（GEOIP CN DIRECT + MATCH PROXY）
  - `MihomoManager`：子进程管理（start / stop / restart / reload）、流量轮询循环、配额检查循环（按客户端 IP 累计流量，超限触发配置热重载）
- **API 路由（9 个模块）**：
  - `/api/auth` —— 登录 / 刷新 / 当前用户
  - `/api/system` —— 内核启停 / 模式切换 / 日志级别 / 状态查询
  - `/api/subscriptions` —— CRUD + 立即更新 + 批量更新
  - `/api/proxies` —— 节点列表 / 分组 / 选择节点 / 测速
  - `/api/rules` —— CRUD（写操作触发配置热重载）
  - `/api/connections` —— 列表 / 关闭单个 / 关闭全部
  - `/api/traffic` —— 实时流量 / 内存 / 历史统计（可重置）
  - `/api/limits` —— CRUD（global / group / client 三种粒度）
  - `/api/quotas` —— CRUD + 重置使用量

#### 前端（Vue 3 + Pinia + reka-ui）

- **黑金主题**：Tailwind CSS + CSS 变量（gold #d4af37 + ink 深黑）
- **9 个页面**：
  - LoginView —— 全屏登录卡片
  - DashboardView —— 4 卡片概览 + 流量统计表 + 2 秒轮询
  - ProxiesView —— 分组卡片 + 节点表格 + 测速
  - SubscriptionsView —— 订阅 CRUD + 立即更新 / 批量更新
  - RulesView —— 规则 CRUD + 类型 / 目标选择器
  - LimitsView —— 限速 CRUD + scope 选择 + formatSpeed 预览
  - QuotasView —— 配额 CRUD + 使用率进度条 + 重置
  - ConnectionsView —— 连接列表 + 3 秒轮询 + 自动刷新开关 + 关闭连接
  - SettingsView —— 内核启停 + 模式切换 + 日志级别
- **认证**：Pinia Setup Store + localStorage 持久化，axios 请求拦截器注入 Bearer token，401 自动跳转登录
- **组件库**：完整 shadcn-vue（reka-ui）组件库 + ConfirmDialog + Toaster
- **路由守卫**：未登录自动跳转登录页，登录后回跳原路径

#### 部署与运维

- **Docker 多阶段构建**：Node 20 构建前端 → Python 3.12-slim 运行时 + 内置 mihomo 二进制（支持 build-arg 指定版本 / 架构 / 下载代理）
- **docker-compose.yml**：暴露 8000 (Web) + 7890 (代理混合端口)，数据卷持久化
- **Jenkinsfile**：符合 project-release.md 规范，sshPublisher + docker run 方式部署到 hongkong (38.92.9.207)
- **启动脚本**：start.bat（端口检查 + 前端构建检查 + mihomo 二进制检查 + 启动后端）+ kill_stale_port.ps1（本应用占用则杀掉重启，他人占用则提示）
